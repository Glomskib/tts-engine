# TikTok Studio Upload

Browser automation script that uploads videos to TikTok Studio from a local upload-pack folder. Uses Playwright headed mode with storageState persistence for login.

## Prerequisites

- Node.js 20+
- Playwright: `npx playwright install chromium`
- A TikTok account with TikTok Studio access
- An upload-pack folder (generated by `publish:pack` or manually assembled)

## Usage

```bash
cd /Users/brandonglomski/tts-engine/web

# Upload from pack directory (saves as draft by default)
npx tsx scripts/publish/tiktok-studio-upload.ts ~/FlashFlowUploads/2026-02-21/skeptic/product-slug

# Post immediately instead of draft
npx tsx scripts/publish/tiktok-studio-upload.ts ~/FlashFlowUploads/2026-02-21/skeptic/product-slug --post

# Dry run — print pack contents + check TikTok Studio selectors
npx tsx scripts/publish/tiktok-studio-upload.ts ~/FlashFlowUploads/2026-02-21/skeptic/product-slug --dry-run
```

## Flags

| Flag | Description |
|------|-------------|
| `--post` | Publish immediately (default: save as draft) |
| `--dry-run` | Print pack contents, open browser, verify selectors. No upload. |

## Upload Pack Directory

The script reads these files from the pack folder:

```
<pack-dir>/
├── caption.txt       # Caption text (required)
├── hashtags.txt      # Hashtags, space or newline separated (required)
├── video.mp4         # Video file (required, or provide video_url in metadata.json)
├── metadata.json     # { product: { tiktok_product_id }, video_url, video_id } (optional)
├── product.txt       # Fallback product ID: "TikTok Product ID: <id>" (optional)
└── cover.txt         # Cover/thumbnail text (read but not uploaded — TikTok auto-generates)
```

**Product ID resolution order:**
1. `metadata.json` → `product.tiktok_product_id`
2. `metadata.json` → `product_id`
3. `product.txt` → line matching `TikTok Product ID: <id>`, or first line

**Video resolution order:**
1. `video.mp4` in pack directory
2. Any `.mp4` file in pack directory
3. `metadata.json` → `video_url` (downloaded to pack dir)
4. `metadata.json` → `video_source.google_drive_url` (converted to direct download)

## Login Persistence

Session is stored at `~/.flashflow/tiktok-studio.storageState.json`.

**First run:** The script opens a headed browser, detects you're not logged in, and pauses with a prompt. Log in manually in the browser window, then press Enter. The session is saved automatically.

**Subsequent runs:** The saved storageState is loaded into the browser context. If the session expires, the script pauses again for re-login.

## Steps Performed

1. Open TikTok Studio upload page (headed Chromium)
2. Check login — pause for manual login if needed, save storageState
3. Upload video file via hidden `<input type="file">`
4. Fill description field (caption + hashtags) in contenteditable editor
5. Open product link selector, search by `tiktok_product_id`, select first result
6. Save as draft (default) or click Post (`--post` flag)

## Dry Run

`--dry-run` opens the browser and checks whether key UI selectors are present:

- File input (`input[type="file"]`)
- Caption editor (`[contenteditable="true"]`)
- Add product button (`button:has-text("Add product")`)
- Draft button (`button:has-text("Save as draft")`)
- Post button (`button:has-text("Post")`)

Use this to verify TikTok Studio hasn't changed its UI before a real upload.

## Architecture

This script (`web/scripts/publish/tiktok-studio-upload.ts`) is a CLI entry point that reuses step-function modules from `skills/tiktok-studio-uploader/`:

```
skills/tiktok-studio-uploader/
  upload.ts       → uploadVideoFile(page, videoPath)
  description.ts  → fillDescription(page, description)
  product.ts      → attachProductByID(page, productId)
  draft.ts        → saveDraft(page) / publishPost(page)
  selectors.ts    → all TikTok Studio element selectors
```

The script provides its own browser management using storageState (vs. the persistent-context approach in the skill's `browser.ts`).

## Limitations

- **Selector fragility:** TikTok Studio UI changes may break selectors. Run `--dry-run` first.
- **No CAPTCHA bypass:** User resolves captchas manually in the headed browser.
- **Cover image:** Not automated (TikTok auto-generates thumbnails).
- **Drafts are device-local:** TikTok drafts are only visible on the device where they were created.
